# BMAD Project

A Business Model Application Development (BMAD) project structure.

## Directory Structure

```
test-bmad-project/
├── _bmad/              # BMAD methodology documents and artifacts
├── _bmad-output/       # Generated outputs from BMAD processes
├── src/                # Source code for the application
├── tests/              # Test files and test suites
├── docs/               # Project documentation
└── README.md           # This file
```

## Purpose of Each Directory

### `_bmad/`
Contains BMAD methodology documents including:
- Business requirements
- Process specifications
- Methodology templates
- Planning artifacts

### `_bmad-output/`
Generated outputs from BMAD processes:
- Generated documentation
- Analysis results
- Process outputs
- Reports

### `src/`
Application source code:
- Main application logic
- Modules and components
- Configuration files
- Entry points

### `tests/`
Test files and test suites:
- Unit tests
- Integration tests
- Test fixtures
- Test configurations

### `docs/`
Project documentation:
- API documentation
- User guides
- Technical specifications
- Change logs

## Getting Started

1. Add BMAD methodology documents to `_bmad/`
2. Run BMAD processes to generate outputs in `_bmad-output/`
3. Implement application code in `src/`
4. Write tests in `tests/`
5. Maintain documentation in `docs/`
